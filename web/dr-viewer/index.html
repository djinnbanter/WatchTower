Watchtower disaster recovery

Server won’t start?
We’ll help you fix it.

Two quick steps — run one command on your host, then drop the zip here. No running mod required.

  1. 1

    Run this on your game server

    SSH into the host (the machine that runs Minecraft). Open a terminal in your server’s mods folder.

    your host 
    # go to your server mods folder
cd /path/to/your/server/mods

# build the recovery zip (one file)
java -jar watchtower-cli-1.0.0.jar dr

    Where does the zip go?

    • Normal: saved in the folder you ran the command from — usually mods/.
    • Look for: watchtower-dr-bundle-<timestamp>.zip — that’s the only file you need.
    My panel won’t let me save files here Click to show the --out /tmp workaround

    Add --out /tmp to write on the host, not inside your Minecraft server folder:

    java -jar watchtower-cli-1.0.0.jar dr --out /tmp

    /tmp is the Linux temp folder on the same machine as your SSH session. Grab the zip from /tmp with SFTP or your panel’s file manager. On Windows hosts, try --out %TEMP% instead.

  2. 2

    Upload that zip here

    We’ll read the crash loop and show you what to fix first.
